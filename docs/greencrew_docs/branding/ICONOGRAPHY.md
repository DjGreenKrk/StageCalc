# GreenCrew Tools Iconography

## Cel dokumentu

Ten dokument opisuje zasady tworzenia ikon dla aplikacji GreenCrew Tools.

Ikony powinny być spójne między aplikacjami i łatwe do rozpoznania nawet w małych rozmiarach.

---

# 1. Główna zasada

Każda aplikacja GreenCrew Tools używa wspólnej bazy:

**geometryczny heksagon GreenCrew + jeden symbol aplikacji.**

---

# 2. Heksagon

Heksagon jest elementem wspólnym całej rodziny aplikacji.

Nie należy:

- zmieniać jego charakteru,
- zastępować go inną figurą,
- dodawać ozdobnych ramek,
- tworzyć realistycznych efektów 3D.

Można:

- lekko dopasować grubość linii,
- dopasować symbol wewnętrzny,
- użyć wersji pełnej lub konturowej zgodnie z projektem.

---

# 3. Symbol aplikacji

Każda aplikacja ma jeden główny symbol.

Przykłady:

| Aplikacja | Symbol |
|---|---|
| WorkLog | stoper |
| StageCalc | błyskawica |
| CableListTool | kabel lub złącze |
| GetOnTime Signage | ekran |
| MultiTimer | zegar |
| CrewList | grupa / identyfikator / lista osób |

Nie należy łączyć wielu symboli w jednej ikonie.

---

# 4. Styl ikon

Preferowany styl:

- geometryczny,
- prosty,
- techniczny,
- nowoczesny,
- minimalistyczny.

Unikać:

- realizmu,
- ilustracji,
- wielu detali,
- gradientów bez potrzeby,
- stylu kreskówkowego,
- symboli stockowych bez dopasowania.

---

# 5. Czytelność

Ikona musi być czytelna w rozmiarach:

- 32×32 px,
- 48×48 px,
- 128×128 px,
- 512×512 px,
- 1024×1024 px.

Drobne detale mogą zanikać przy 32×32 px, ale główny symbol musi pozostać rozpoznawalny.

---

# 6. Tło

Preferowane tło:

- przezroczyste dla plików źródłowych,
- czarne dla splash screena,
- zgodne z wymaganiami platformy dla ikon adaptacyjnych Androida.

---

# 7. Kolory

Podstawowy kolor ikon:

`#00C853`

Dopuszczalne odcienie:

`#00E676`

`#00B248`

Tło:

`#000000`

Ikony nie powinny używać wielu kolorów aplikacji.

---

# 8. Ikony w interfejsie

Preferowana biblioteka:

Material Icons / Material Symbols.

Aktualnie dopuszczalne:

- `MaterialIcons-Regular`,
- Material Symbols Rounded,
- Material Symbols Outlined.

Nie mieszać wielu rodzin ikon bez uzasadnienia.

---

# 9. Zasada znaczenia

Ikona powinna przedstawiać funkcję, nie ozdobę.

Przykład:

Dla eksportu użyć ikony eksportu / pliku.

Nie używać losowej grafiki tylko dlatego, że wygląda technicznie.

---

# 10. Ikony statusów

Statusy powinny być wspierane ikoną i kolorem.

Przykłady:

- sukces: check,
- błąd: error / close,
- ostrzeżenie: warning,
- informacja: info,
- offline: cloud_off / wifi_off.

Nie przekazywać informacji wyłącznie kolorem.

---

# 11. StageCalc

Symbol główny:

**błyskawica**

Znaczenie:

- zasilanie,
- moc,
- przepływ energii,
- dystrybucja,
- kontrola obciążeń.

Unikać:

- znaku wysokiego napięcia BHP,
- żółtego komiksowego pioruna,
- realistycznych przewodów.

---

# 12. WorkLog

Symbol główny:

**stoper**

Znaczenie:

- czas pracy,
- rejestracja,
- pomiar,
- rozliczenie.

Unikać:

- kalendarzy jako głównego symbolu,
- clipboardów,
- ikon HR.

---

# 13. CableListTool

Symbol główny:

**kabel / złącze**

Znaczenie:

- połączenia,
- przebiegi,
- oznaczenia,
- dokumentacja kablowa.

Unikać:

- wielu różnych złączy jednocześnie,
- realistycznych przewodów,
- zbyt szczegółowych schematów.

---

# 14. Checklist dla nowej ikony

Przed zaakceptowaniem ikony sprawdzić:

- czy zawiera heksagon GreenCrew,
- czy ma jeden czytelny symbol,
- czy działa w 32×32 px,
- czy działa na czarnym tle,
- czy nie jest pstrokata,
- czy nie wygląda jak aplikacja lifestyle,
- czy pasuje do pozostałych aplikacji.
