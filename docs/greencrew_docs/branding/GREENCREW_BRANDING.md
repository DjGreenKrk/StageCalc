# GreenCrew Tools Branding Standard

## Cel dokumentu

Ten dokument definiuje wspólny branding dla wszystkich aplikacji należących do ekosystemu **GreenCrew Tools**.

Każdy nowy projekt powinien zawierać ten plik bez modyfikacji oraz własny plik `BRANDING.md` opisujący wyłącznie cechy konkretnej aplikacji.

---

# O ekosystemie

GreenCrew Tools to rodzina profesjonalnych aplikacji tworzonych przez GreenCrew dla branży technicznej, scenicznej, instalacyjnej i serwisowej.

Aplikacje mają być:

- szybkie,
- praktyczne,
- czytelne,
- offline first,
- profesjonalne,
- stonowane wizualnie,
- przydatne w realnej pracy terenowej.

Nie są tworzone jako aplikacje lifestyle, HR, wellness, ERP ani typowe aplikacje produktywności.

**Narzędzia stworzone przez technika dla techników.**

# Grupa docelowa

Aplikacje GreenCrew Tools są tworzone dla:

- techników scenicznych,
- elektryków,
- realizatorów,
- monterów,
- serwisantów,
- operatorów AV,
- osób pracujących podczas wydarzeń,
- osób pracujących w terenie.

Projektując aplikacje należy zakładać, że użytkownik może:

- działać pod presją czasu,
- mieć brudne ręce,
- pracować w ciemności,
- pracować na małym ekranie,
- nie mieć Internetu,
- potrzebować szybkiego dostępu do danych.

---

# Charakter marki

Aplikacje powinny sprawiać wrażenie cyfrowych odpowiedników profesjonalnych narzędzi technicznych.

Inspiracje:

- miernik instalacyjny,
- tester kabli,
- kalkulator techniczny,
- panel operatorski,
- sterownik przemysłowy,
- dokumentacja techniczna,
- aplikacja serwisowa.

Unikać stylistyki:

- aplikacji wellness,
- aplikacji lifestyle,
- aplikacji HR,
- korporacyjnych paneli zarządzania,
- kreskówek,
- ozdobnych ilustracji,
- motywów ekologicznych mimo nazwy GreenCrew.

# Identyfikacja wizualna

## Element wspólny

Każda aplikacja korzysta z motywu geometrycznego heksagonu GreenCrew.

Heksagon jest elementem rozpoznawczym rodziny aplikacji.

## Element indywidualny

Każda aplikacja posiada jeden własny symbol umieszczony wewnątrz heksagonu.

Przykłady:

| Aplikacja | Symbol |
|---|---|
| GreenCrew | pusty heksagon |
| WorkLog | stoper |
| StageCalc | błyskawica |
| CableListTool | kabel lub złącze |
| GetOnTime Signage | ekran |
| GetOnTime MultiTimer | zegar |
| CrewList | grupa / identyfikator / lista osób |

Nie należy tworzyć osobnych znaków całkowicie oderwanych od heksagonu.

# Zasada koloru

Kolor nie służy do dekoracji.

Kolor służy do:

- wskazania akcji,
- oznaczenia stanu,
- zwrócenia uwagi,
- pokazania błędu,
- pokazania ostrzeżenia,
- pokazania sukcesu.

Interfejs powinien być głównie ciemny, neutralny i stonowany.

Dopuszczalny jest jeden główny akcent aplikacji.

Domyślnym akcentem jest GreenCrew Green.

# Splash screen

Standard splash screena:

- czarne tło,
- wyśrodkowana ikona aplikacji,
- brak sloganów,
- brak dodatkowych logotypów,
- brak własnych animacji, o ile platforma ich nie wymaga.

- nazwa aplikacji,

- wersja,
- autor,
- organizacja,
- strona,

- licencja,
